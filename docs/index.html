Gestionnaire de Stock de Magasin

Application Java Swing pour gerer le stock d'un magasin : produits, fournisseurs, commandes et utilisateurs. Authentification par role, donnees stockees dans le cloud via une API REST.

Equipe

Projet realise par :

  • Mohamed Rayen Sbai
  • Ranim Belguith

Ce que fait l'application

Un magasin a besoin de suivre ce qu'il a en stock, qui le fournit et ce que les clients commandent. Notre application repond a ces besoins avec quatre ecrans principaux :

ModuleOperationsAcces
ProduitsAjouter, modifier, supprimer, listerTout role
FournisseursAjouter, modifier, supprimer, listerTout role
CommandesEnregistrer, modifier, listerTout role
UtilisateursAjouter, modifier, supprimer, listerAdministrateur uniquement

Pile technique

Cote client

  • Java 17
  • Swing pour l'interface
  • FlatLaf (theme IntelliJ) pour le look moderne
  • Gson pour le JSON
  • jBCrypt pour les mots de passe

Cote serveur

  • Appwrite Cloud comme base de donnees
  • API REST (pas de SDK Java officiel)
  • HTTPS, authentification par cle API

Choix de conception

Pourquoi Swing et pas JavaFX ? Le sujet du projet demandait Swing. On a garde Swing mais on l'a habille avec FlatLaf pour eviter l'apparence vieillotte par defaut.

Pourquoi Appwrite et pas MySQL ? On voulait apprendre a consommer une API REST depuis Java. MySQL aurait demande JDBC. Appwrite nous a oblige a manipuler du JSON et a structurer un client HTTP propre, ce qui ressemble plus a un cas reel d'integration.

Pourquoi BCrypt ? Les mots de passe en clair sont une faille triviale. BCrypt est un standard reconnu, avec sel automatique et cout configurable. La librairie jBCrypt fait moins de 100 ko.
