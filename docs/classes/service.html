Services et appels Appwrite

La couche service traduit les operations metier (lister, ajouter, modifier, supprimer) en appels HTTP. Tout passe par ClientAppwrite qui encapsule la signature des requetes.

ClientAppwrite

Fichier : src/main/java/com/magasin/service/ClientAppwrite.java

C'est la classe la plus technique du projet. Elle utilise java.net.http.HttpClient (disponible depuis Java 11) plutot qu'une bibliotheque externe.

Champ statique HTTP

private static final HttpClient HTTP = HttpClient.newBuilder()
        .connectTimeout(Duration.ofSeconds(15))
        .build();

Un seul client HTTP est cree pour toute l'application. Reutiliser le client permet la mise en cache des connexions et evite d'ouvrir un nouveau socket a chaque requete.

Construction d'une requete

private static HttpRequest.Builder requeteBase(String chemin) {
    URI uri = URI.create(Configuration.pointTerminaison() + chemin);
    return HttpRequest.newBuilder()
            .uri(uri)
            .timeout(Duration.ofSeconds(30))
            .header("X-Appwrite-Project", Configuration.idProjet())
            .header("X-Appwrite-Key", Configuration.cleApi())
            .header("Content-Type", "application/json")
            .header("Accept", "application/json");
}

Toutes les requetes Appwrite portent ces trois en-tetes :

  • X-Appwrite-Project identifie le projet cible.
  • X-Appwrite-Key authentifie le client. Notre cle a les scopes tables.* et rows.*.
  • Content-Type et Accept annoncent du JSON dans les deux sens.

Lister les lignes d'une table

public static JsonArray listerLignes(String tableId, List<String> requetes)
        throws AppwriteException {
    StringBuilder url = new StringBuilder(basePourTable(tableId));
    if (requetes != null && !requetes.isEmpty()) {
        url.append("?");
        for (int i = 0; i < requetes.size(); i++) {
            if (i > 0) url.append("&");
            url.append("queries[]=").append(URLEncoder.encode(
                    requetes.get(i), StandardCharsets.UTF_8));
        }
    }
    HttpRequest req = requeteBase(url.toString()).GET().build();
    JsonObject rep = envoyer(req);
    if (rep.has("rows") && rep.get("rows").isJsonArray())
        return rep.getAsJsonArray("rows");
    if (rep.has("documents") && rep.get("documents").isJsonArray())
        return rep.getAsJsonArray("documents");
    return new JsonArray();
}

Points a noter :

  • Les requetes (filtres) sont passees dans le parametre queries[] en URL-encode pour echapper les guillemets et caracteres speciaux.
  • La reponse contient un tableau rows dans la nouvelle API ; on garde la compatibilite avec documents au cas ou.

Creer, mettre a jour, supprimer

public static JsonObject creerLigne(String tableId, JsonObject donnees) {
    JsonObject corps = new JsonObject();
    corps.addProperty("rowId", "unique()");   // demande un id auto a Appwrite
    corps.add("data", donnees);
    return envoyer(requeteBase(basePourTable(tableId))
            .POST(BodyPublishers.ofString(corps.toString(), UTF_8))
            .build());
}

La cle rowId vaut "unique()" : Appwrite genere alors un identifiant aleatoire. Le payload reel est niche sous la cle data.

Methode envoyer et gestion des erreurs

private static JsonObject envoyer(HttpRequest req) throws AppwriteException {
    try {
        HttpResponse<String> rep = HTTP.send(req, BodyHandlers.ofString(UTF_8));
        int code = rep.statusCode();
        if (code >= 200 && code < 300) {
            return JsonParser.parseString(rep.body()).getAsJsonObject();
        }
        // Sinon : on extrait le message d'erreur Appwrite
        String msg = "Erreur HTTP " + code;
        try {
            JsonObject err = JsonParser.parseString(rep.body()).getAsJsonObject();
            if (err.has("message")) msg += " — " + err.get("message").getAsString();
        } catch (Exception ignore) { }
        throw new AppwriteException(msg, code);
    } catch (Exception e) {
        throw new AppwriteException("Echec reseau : " + e.getMessage(), -1);
    }
}

Quand Appwrite renvoie un code 4xx ou 5xx, le corps de reponse est en JSON avec une cle message. On extrait ce message pour que la vue affiche une erreur lisible (par exemple "Invalid query syntax").

AppwriteException

Fichier : src/main/java/com/magasin/service/AppwriteException.java

public class AppwriteException extends Exception {
    private final int codeHttp;
    public AppwriteException(String message, int codeHttp) {
        super(message);
        this.codeHttp = codeHttp;
    }
    public int getCodeHttp() { return codeHttp; }
}

Une exception verifiee (extends Exception) pour forcer les services et controleurs a la gerer explicitement. Le code HTTP est conserve si on a besoin de distinguer 401 (mauvaise cle) de 404 (ligne introuvable).

ServiceProduit

Fichier : src/main/java/com/magasin/service/ServiceProduit.java

Ce service contient aussi des helpers static reutilises par les autres services : extraireId, optString, optInt, optDouble.

Lister tous les produits

public static List<Produit> listerTout() throws AppwriteException {
    JsonArray lignes = ClientAppwrite.listerLignes(table(), Collections.emptyList());
    List<Produit> resultats = new ArrayList<>();
    for (JsonElement el : lignes) {
        resultats.add(depuisJson(el.getAsJsonObject()));
    }
    return resultats;
}

Conversion JSON ↔ Produit

private static JsonObject versJson(Produit p) {
    JsonObject o = new JsonObject();
    o.addProperty("nom", p.getNom());
    o.addProperty("quantite", p.getQuantite());
    o.addProperty("prix", p.getPrix());
    o.addProperty("fournisseur", p.getFournisseur() == null ? "" : p.getFournisseur());
    return o;
}

private static Produit depuisJson(JsonObject o) {
    Produit p = new Produit();
    p.setId(extraireId(o));
    p.setNom(optString(o, "nom"));
    p.setQuantite(optInt(o, "quantite"));
    p.setPrix(optDouble(o, "prix"));
    p.setFournisseur(optString(o, "fournisseur"));
    return p;
}

Helpers utilises :

static String optString(JsonObject o, String k) {
    return o.has(k) && !o.get(k).isJsonNull() ? o.get(k).getAsString() : "";
}
static int optInt(JsonObject o, String k) {
    return o.has(k) && !o.get(k).isJsonNull() ? o.get(k).getAsInt() : 0;
}
static String extraireId(JsonObject o) {
    if (o.has("$id")) return o.get("$id").getAsString();
    if (o.has("id"))  return o.get("id").getAsString();
    return null;
}

L'identifiant de la ligne est sous la cle $id (le $ indique un champ systeme dans Appwrite). On cherche id en secours, au cas ou.

ServiceCommande

C'est le service le plus interessant. L'enregistrement et la modification d'une commande doivent verifier le stock disponible.

Enregistrer une commande

public static Commande ajouter(Commande c) throws AppwriteException {
    Produit p = ServiceProduit.obtenirParId(c.getProduitId());
    if (p == null) {
        throw new AppwriteException("Produit introuvable.", -1);
    }
    if (c.getQuantite() > p.getQuantite()) {
        throw new AppwriteException(
            "Stock insuffisant pour \"" + p.getNom() + "\". "
            + "Disponible : " + p.getQuantite()
            + ", demande : " + c.getQuantite() + ".", -1);
    }
    p.setQuantite(p.getQuantite() - c.getQuantite());
    ServiceProduit.modifier(p);                       // on debite le stock

    JsonObject rep = ClientAppwrite.creerLigne(table(), versJson(c));
    return depuisJson(rep);
}

L'ordre des operations compte. On verifie le stock avant tout. Si on debitait sans verifier, on aurait des stocks negatifs. On modifie le produit avant de creer la commande pour que, si l'ecriture du produit echoue, on n'ait pas une commande orpheline qui pointe vers un stock incorrect.

Modifier une commande

public static Commande modifier(Commande nouvelle) throws AppwriteException {
    Commande ancienne = obtenirParId(nouvelle.getId());

    if (ancienne.getProduitId().equals(nouvelle.getProduitId())) {
        // Meme produit : on applique juste la difference au stock
        int delta = nouvelle.getQuantite() - ancienne.getQuantite();
        if (delta != 0) {
            Produit p = ServiceProduit.obtenirParId(nouvelle.getProduitId());
            if (delta > 0 && p.getQuantite() < delta) {
                throw new AppwriteException("Stock insuffisant...", -1);
            }
            p.setQuantite(p.getQuantite() - delta);
            ServiceProduit.modifier(p);
        }
    } else {
        // Produit different : on rembourse l'ancien produit, on debite le nouveau
        Produit ancien  = ServiceProduit.obtenirParId(ancienne.getProduitId());
        Produit nouveau = ServiceProduit.obtenirParId(nouvelle.getProduitId());

        if (nouveau.getQuantite() < nouvelle.getQuantite()) {
            throw new AppwriteException("Stock insuffisant...", -1);
        }
        ancien.setQuantite(ancien.getQuantite() + ancienne.getQuantite());
        ServiceProduit.modifier(ancien);
        nouveau.setQuantite(nouveau.getQuantite() - nouvelle.getQuantite());
        ServiceProduit.modifier(nouveau);
    }
    JsonObject rep = ClientAppwrite.mettreAJourLigne(table(), nouvelle.getId(), versJson(nouvelle));
    return depuisJson(rep);
}

Deux cas a distinguer :

  • Le client change juste la quantite. On calcule la difference (positive ou negative) et on l'applique au stock.
  • Le client change carrement de produit. On rembourse l'ancien produit (ajout) puis on debite le nouveau (soustraction). Si le nouveau n'a pas assez de stock, on echoue avant de toucher quoi que ce soit.
Limite connue : il n'y a pas de transaction reelle. Si l'ecriture de la commande echoue apres avoir debite le produit, le stock reste decremente. Une vraie application utiliserait une fonction serveur Appwrite (ou une transaction SQL si on etait en MySQL). Pour un projet d'examen, l'approche reste acceptable.

ServiceUtilisateur

Trois operations specifiques : recherche par nom d'utilisateur, authentification, et l'ajout / modification avec hachage du mot de passe.

public static Utilisateur trouverParUsername(String username) throws AppwriteException {
    // Filtre cote client : la base contient peu d'utilisateurs.
    // On evite ainsi un piege de syntaxe sur les filtres TablesDB.
    for (Utilisateur u : listerTout()) {
        if (u.getUsername() != null && u.getUsername().equalsIgnoreCase(username)) {
            return u;
        }
    }
    return null;
}

public static Utilisateur authentifier(String username, String motDePasseClair)
        throws AppwriteException {
    Utilisateur u = trouverParUsername(username);
    if (u == null) return null;
    if (!GestionnaireMotDePasse.verifier(motDePasseClair, u.getPassword())) return null;
    return u;
}

L'authentification renvoie l'utilisateur si tout est bon, null sinon. La vue distingue les deux cas pour afficher "Identifiants incorrects".

Pour la creation / mise a jour d'un utilisateur :

public static Utilisateur ajouter(String username, String motDePasseClair, String role) {
    JsonObject d = new JsonObject();
    d.addProperty("username", username);
    d.addProperty("password", GestionnaireMotDePasse.hacher(motDePasseClair));  // BCrypt
    d.addProperty("role", role);
    return depuisJson(ClientAppwrite.creerLigne(table(), d));
}

Le mot de passe en clair ne quitte jamais la machine du client : on le passe a BCrypt avant de l'envoyer dans le JSON.
