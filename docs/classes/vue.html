Vues Swing

L'interface utilise Swing avec le look FlatLaf. Les fenetres principales sont la connexion et le tableau de bord. Chaque module a son propre panneau et son propre dialogue de saisie.

Application (point d'entree)

Fichier : src/main/java/com/magasin/Application.java

public static void main(String[] args) {
    // Mise a l'echelle pour ecrans haute densite
    System.setProperty("sun.java2d.uiScale.enabled", "true");
    if (System.getProperty("sun.java2d.uiScale") == null) {
        System.setProperty("sun.java2d.uiScale",
            String.valueOf(detecterEchelle()));
    }
    // Anti-aliasing pour les polices
    System.setProperty("awt.useSystemAAFontSettings", "on");
    System.setProperty("swing.aatext", "true");

    // Nom de l'application au niveau du serveur de fenetres
    try {
        Toolkit tk = Toolkit.getDefaultToolkit();
        Field f = tk.getClass().getDeclaredField("awtAppClassName");
        f.setAccessible(true);
        f.set(tk, "MagasinPro");
    } catch (Exception ignore) {}

    SwingUtilities.invokeLater(() -> {
        initialiserApparence();        // FlatLaf
        Configuration.charger();        // config.properties
        new FenetreConnexion().setVisible(true);
    });
}

Deroulement :

  1. On positionne les proprietes systeme avant d'instancier le moindre composant AWT. Sinon, elles seraient ignorees.
  2. On essaie de definir awtAppClassName a "MagasinPro". Ca donne un nom propre dans la barre des taches.
  3. On bascule sur l'EDT via SwingUtilities.invokeLater pour creer la premiere fenetre. Toute creation de composant Swing doit se faire sur ce thread.

Initialisation de l'apparence

private static void initialiserApparence() throws Exception {
    UIManager.put("Button.arc", 12);            // boutons arrondis
    UIManager.put("Component.arc", 10);
    UIManager.put("TextComponent.arc", 10);     // champs texte arrondis
    UIManager.put("Component.accentColor", COULEUR_ACCENT);
    UIManager.put("Table.rowHeight", 32);
    UIManager.put("Table.showHorizontalLines", true);
    UIManager.put("defaultFont", new Font("Segoe UI", Font.PLAIN, 13));
    FlatIntelliJLaf.setup();
}

FlatLaf accepte beaucoup de proprietes UIManager qui changent les coins, les marges et les couleurs sans avoir a deriver les composants. Les constantes COULEUR_ACCENT, COULEUR_SUCCES et autres sont definies en haut du fichier.

FenetreConnexion

Fichier : src/main/java/com/magasin/vue/FenetreConnexion.java

Une JFrame divisee en deux : un panneau a gauche avec un degrade et un titre, un panneau a droite avec le formulaire de connexion.

public FenetreConnexion() {
    super("Gestionnaire de Stock — Connexion");
    setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);

    // Taille adaptative : 75% de l'ecran, plafonnee
    Dimension ecran = Toolkit.getDefaultToolkit().getScreenSize();
    int l = Math.min(960, (int) (ecran.width  * 0.75));
    int h = Math.min(620, (int) (ecran.height * 0.80));
    setSize(l, h);
    setMinimumSize(new Dimension(Math.min(700, l), Math.min(500, h)));
    setLocationRelativeTo(null);
    construire();
}

On limite la taille au minimum entre une valeur fixe (960×620) et un pourcentage de l'ecran. Sur un petit ecran l'app reste utilisable. Sur grand ecran on ne s'etale pas au-dela du raisonnable.

tenterConnexion

private void tenterConnexion() {
    String u = champUsername.getText().trim();
    String p = new String(champMotDePasse.getPassword());
    etiquetteMessage.setText(" ");
    ControleurConnexion.connecter(this, u, p, message -> {
        etiquetteMessage.setText(message);
        etiquetteMessage.setForeground(Application.COULEUR_DANGER);
    });
}

On delegue au controleur. Le quatrieme parametre est un Consumer<String> que le controleur appelle en cas d'erreur (saisie invalide, identifiants incorrects, erreur reseau). La vue ne sait pas pourquoi c'est echoue, elle se contente d'afficher le message.

Panneaux CRUD

Les quatre panneaux suivent la meme structure :

  1. Un en-tete avec un titre et une description (EnTetePanneau).
  2. Une barre d'actions (boutons Ajouter / Modifier / Supprimer / Rafraichir).
  3. Une JTable stylisee qui affiche les donnees.

PanneauProduits

private final DefaultTableModel modele = new DefaultTableModel(
        new Object[]{"ID", "Nom", "Quantite", "Prix", "Fournisseur"}, 0) {
    @Override public boolean isCellEditable(int r, int c) { return false; }
};
private final JTable tableau = new TableurStylise(modele);

On surcharge isCellEditable pour rendre le tableau en lecture seule. La modification passe par le dialogue, pas par la cellule.

Action "Ajouter"

private void ajouter() {
    Produit p = DialogueProduit.afficher(this, null);   // null = creation
    if (p == null) return;                              // l'utilisateur a annule
    executerService(() -> ControleurProduits.ajouter(p), "Produit ajoute.");
}

DialogueProduit.afficher retourne null si l'utilisateur a clique sur Annuler ou ferme la fenetre. Sinon il retourne un nouveau Produit rempli.

Action "Modifier"

private void modifier() {
    Produit p = produitSelectionne();
    if (p == null) {
        GestionnaireErreurs.erreur(this, "Veuillez selectionner un produit.");
        return;
    }
    Produit modifie = DialogueProduit.afficher(this, p);   // passe l'existant
    if (modifie == null) return;
    modifie.setId(p.getId());                              // on conserve l'id
    executerService(() -> ControleurProduits.modifier(modifie), "Produit modifie.");
}

On verifie qu'une ligne est selectionnee, on ouvre le dialogue prerempli, puis on remet l'id avant d'appeler le service (le dialogue ne le manipule pas).

Helper executerService

private void executerService(Action action, String messageSucces) {
    new SwingWorker<Object, Void>() {
        protected Object doInBackground() throws Exception {
            return action.executer();
        }
        protected void done() {
            try {
                get();                      // releve l'exception eventuelle
                rafraichir();
                GestionnaireErreurs.information(this, messageSucces);
            } catch (Exception ex) {
                GestionnaireErreurs.erreur(this, "Operation echouee", ex.getCause());
            }
        }
    }.execute();
}

Le meme schema est repete dans les quatre panneaux : SwingWorker, refresh apres succes, dialogue d'erreur sinon.

PanneauCommandes : combo des produits

Pour enregistrer une commande, l'utilisateur choisit un produit dans un combo. Le panneau garde un cache des produits :

private List<Produit> produitsCache = List.of();
private final Map<String, String> nomParId = new HashMap<>();

public void rafraichir() {
    new SwingWorker<Object[], Void>() {
        protected Object[] doInBackground() throws Exception {
            List<Produit> prods = ControleurCommandes.listerProduits();
            List<Commande> cmds = ControleurCommandes.listerTout();
            return new Object[]{prods, cmds};
        }
        protected void done() {
            Object[] res = get();
            produitsCache = (List<Produit>) res[0];
            // Mappe id -> nom pour afficher le nom dans le tableau
            nomParId.clear();
            for (Produit p : produitsCache) nomParId.put(p.getId(), p.getNom());
            // ... remplit le modele
        }
    }.execute();
}

On charge produits et commandes en parallele dans le meme worker. La map nomParId sert a afficher le nom du produit dans le tableau au lieu de son identifiant cryptique.

Dialogues de saisie

Chaque module a son dialogue. Tous suivent le meme schema :

  • Une methode statique afficher(parent, existant) qui ouvre une JDialog modale.
  • Un panneau de formulaire avec GridBagLayout.
  • Une barre d'actions avec deux boutons : Annuler et Valider.
  • Une AtomicReference qui contient le resultat pour le retour de methode.
final AtomicReference<Produit> resultat = new AtomicReference<>();
valider.addActionListener(e -> {
    try {
        String nom = champNom.getText().trim();
        if (nom.isEmpty()) throw new IllegalArgumentException("Le nom est obligatoire.");
        int qte = Integer.parseInt(champQte.getText().trim());
        if (qte < 0) throw new IllegalArgumentException("La quantite doit etre positive.");
        // ...
        resultat.set(new Produit(idExistant, nom, qte, prix, fournisseur));
        dialogue.dispose();
    } catch (NumberFormatException nfe) {
        GestionnaireErreurs.erreur(dialogue, "Quantite ou prix invalide.");
    } catch (IllegalArgumentException iae) {
        GestionnaireErreurs.erreur(dialogue, iae.getMessage());
    }
});

dialogue.setVisible(true);   // bloque jusqu'a la fermeture (modal)
return resultat.get();

Le setVisible(true) sur une dialog modale bloque l'EDT jusqu'a ce que la dialog soit fermee. Au retour, on lit la valeur stockee dans l'AtomicReference.

PanneauGradient

protected void paintComponent(Graphics g) {
    Graphics2D g2 = (Graphics2D) g.create();
    g2.setRenderingHint(RenderingHints.KEY_ANTIALIASING,
                        RenderingHints.VALUE_ANTIALIAS_ON);
    GradientPaint gp = new GradientPaint(0, 0, couleurDebut,
                                          getWidth(), getHeight(), couleurFin);
    g2.setPaint(gp);
    g2.fillRect(0, 0, getWidth(), getHeight());
    g2.dispose();
    super.paintComponent(g);
}

On surcharge paintComponent pour dessiner le degrade en arriere-plan. Le panneau reste opaque et accepte des enfants Swing normaux par-dessus.

PanneauCarte

Un panneau avec coins arrondis et une legere ombre, utilise pour la "carte" centrale de l'ecran de connexion. Memes principes que PanneauGradient : on dessine le fond a la main avant super.paintComponent.

TableurStylise

public class TableurStylise extends JTable {
    public TableurStylise(TableModel modele) {
        super(modele);
        setSelectionMode(ListSelectionModel.SINGLE_SELECTION);
        setRowHeight(34);
        setShowGrid(false);
        setFillsViewportHeight(true);
        setDefaultRenderer(Object.class, new RenduAlterne());
    }
    private static class RenduAlterne extends DefaultTableCellRenderer {
        public Component getTableCellRendererComponent(...) {
            // Couleur alternee : blanc / gris clair
            if (isSelected) c.setBackground(new Color(0xE0E7FF));
            else c.setBackground(row % 2 == 0 ? Color.WHITE : new Color(0xF9FAFB));
            return c;
        }
    }
}

Un tableau qui alterne les couleurs de ligne pour faciliter la lecture, sans bordures intercellulaires. Le rendu de selection utilise une teinte bleue claire.
